# 真實底圖回歸測試 / Real-sheet regression fixtures

把「實際生成的貼圖底圖」原始檔放進這個資料夾，E2E 測試
（`e2e/fixtures.spec.ts`）會自動對每一張圖跑完整流程：上傳 → 綠幕自動
切割 → 驗證切出的張數。合成測試圖能重現已知的失敗模式，但真實底圖才是
最終標準——放在這裡的圖，之後每個 PR 的 CI 都會自動用它回歸。

## 檔名規則

`任意名稱-張數.png`，張數必須是平台支援的數量（8/16/24/32/40）。

例：

- `golden-dogs-40.png` — 5×8、40 張的底圖
- `pigs-16.png` — 4×4、16 張的底圖

支援 `.png` / `.jpg` / `.webp`。請放**原始輸出檔**（從生成結果直接下載
的檔案），不要放聊天軟體或截圖工具壓過的版本——壓縮痕跡本身會改變去背
行為，測起來才和實際使用一致。

## 怎麼上傳（不需要本機 git）

GitHub 網頁 → 本資料夾 → `Add file` → `Upload files` → 拖入圖片 →
commit。下一次 CI 就會帶著它跑。
